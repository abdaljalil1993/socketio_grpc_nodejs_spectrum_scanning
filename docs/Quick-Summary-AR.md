# ملخص سريع: تخزين ملفات IQ

## 🎯 الإجابة المباشرة على السؤال

### "كيف عم يتخزن ملف IQ بقاعدة البيانات؟"

**الإجابة:** لا يتم تخزين الملف IQ **داخل** قاعدة البيانات!

بدلاً من ذلك:

```
┌─────────────────┐          ┌──────────────────┐
│ ملف IQ الخام    │          │ قاعدة البيانات   │
│   (8-500 MB)    │          │   (MySQL)        │
│                 │          │                  │
│ • البيانات      │    ←→    │ • UUID           │
│ • الإشارات      │          │ • الفهرس         │
│ • القياسات      │          │ • البيانات الوصفية│
└─────────────────┘          └──────────────────┘
   في الملفات                   في الجداول
 storage/iq-files/          signal_records table
```

---

### "وهل عملية التخزين صحيحة؟"

**الإجابة:** ✅ **نعم، بشكل عام صحيحة، لكن ليست مثالية**

#### ✅ ما هو صحيح:

1. **معيار SigMF:**
   - استخدام معيار دولي معروف
   - توثيق شامل للبيانات

2. **حماية البيانات:**
   - حساب SHA512 لكل ملف
   - التحقق من سلامة البيانات

3. **معالجة الأخطاء:**
   - إذا فشل الحفظ، يتم حذف كل شيء
   - منع بيانات متضاربة

4. **روابط واضحة:**
   - كل ملف له UUID فريد
   - ربط واضح بين DB والملفات

#### ⚠️ ما الذي ليس مثالياً:

1. **لا حماية من الانقطاع:**
   - إذا انقطع الاتصال أثناء الحفظ → ملفات يتيمة

2. **لا فحوصات دورية:**
   - قد تكون هناك ملفات مفقودة ولا أحد يعرف

3. **لا نسخ احتياطية:**
   - فقدان كل شيء إذا حدث عطل

4. **لا حد للتخزين:**
   - قد يمتلئ القرص بدون تحذير

---

### "ولا عم نروح البيانات فيه؟"

**الإجابة:** ⚠️ **نعم، هناك احتمالية فقدان البيانات في هذه الحالات:**

#### 1️⃣ انقطاع الاتصال أثناء الرفع

```
المستخدم يرفع ملف ← الخادم يبدأ بالحفظ ← انقطع الإنترنت!

النتيجة:
✓ ملف IQ محفوظ في storage/
✗ ملف Metadata لم يُحفظ
✗ السجل لم يُحفظ أو تم حذفه

❌ بيانات ضائعة أو معلقة
```

#### 2️⃣ حذف يدوي للملفات

```
مسؤول يحذف الملف مباشرة من الخادم:
$ rm storage/iq-files/recording-xyz.sigmf-data

النتيجة:
✓ السجل باقٍ في قاعدة البيانات
✗ الملف مفقود

❌ بيانات متضاربة - لا يمكن الوصول إليها
```

#### 3️⃣ عطل الخادم

```
انقطع الكهرباء أثناء كتابة الملف

النتيجة:
✗ ملفات ناقصة أو تالفة
✗ بيانات غير متطابقة

❌ فقدان بيانات كامل
```

#### 4️⃣ امتلاء القرص

```
مساحة التخزين امتلأت

النتيجة:
✗ لا يمكن حفظ ملفات جديدة
❌ توقف النظام بالكامل
```

## ✅ الحل السريع (اليوم)

### خطوة 1: فحص الوضع الحالي

```bash
# عد السجلات
mysql> SELECT COUNT(*) FROM signal_records WHERE iq_file IS NOT NULL;

# عد الملفات
ls storage/iq-files/*.sigmf-data | wc -l

# النسبة يجب تكون متطابقة تقريباً
```

### خطوة 2: تثبيت مراقب دوري

```typescript
// في app.ts
const monitor = new StorageMonitorService({
  intervalMs: 24 * 60 * 60 * 1000, // كل 24 ساعة
  enableIntegrityCheck: true,
  enableStatsReport: true
});

if (process.env.NODE_ENV === 'production') {
  monitor.start();
}
```

